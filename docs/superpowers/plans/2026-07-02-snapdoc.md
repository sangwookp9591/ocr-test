# snapdoc 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** zero-dependency TS 문서 스캐너 라이브러리(detect/warp/quality/scan/highlight)를 `ocr-mvp/snapdoc/`에 만들고 ocr-mvp 웹버전에 통합한다.

**Architecture:** 스펙 `docs/superpowers/specs/2026-07-02-snapdoc-scanner-library-design.md`. detect=순수 TS(360px 축소), warp=WebGPU+CPU 폴백, CV 모듈은 DOM 무접촉(`{data,width,height}`)으로 node 테스트.

**Tech Stack:** TypeScript 5, tsc 빌드(번들러 없음 — ESM 출력 그대로), node:test.

## Global Constraints

- 런타임 의존성 **0개** (devDeps는 typescript만)
- CV 모듈(gray/edge/quad/homography/warp-cpu/blur)은 **DOM API 금지** — `Gray = { data: Float32Array, width, height }` 평면 타입만
- 감지 작업 해상도 **최대 360px**, 워핑 기본 최대 출력 **1600px**
- 커밋 메시지 끝에 `Claude-Session: https://claude.ai/code/session_01G2xtKoL79wUu2C3jskQbeA`

### Task 1: 패키지 스캐폴드 + gray + blur (+테스트)

**Files:** Create `snapdoc/package.json`, `snapdoc/tsconfig.json`, `snapdoc/src/types.ts`, `snapdoc/src/gray.ts`, `snapdoc/src/blur.ts`, `snapdoc/test/gray.test.mjs`, `snapdoc/test/blur.test.mjs`, `snapdoc/.gitignore`

**Interfaces (Produces):**
- `types.ts`: `Gray { data: Float32Array; width: number; height: number }`, `Point {x,y}`, `Quad {tl,tr,br,bl}`
- `gray.ts`: `toGray(rgba: Uint8ClampedArray, w, h): Gray`, `stretchContrast(g: Gray): Gray` (2%~98% 백분위 클리핑, in-place)
- `blur.ts`: `blurScore(g: Gray): number` (라플라시안 분산 — 높을수록 선명)

**Steps:** package.json(`"type":"module"`, scripts: `build: tsc`, `test: npm run build && node --test test/`) → 실패 테스트 작성(그레이 변환 기지값, 대비 스트레칭 후 min/max, 선명vs블러 합성 쌍 blurScore 대소) → 구현 → `npm test` 통과 → 커밋 `feat(snapdoc): scaffold with gray conversion and blur score`

### Task 2: edge + quad (+테스트) — 감지 코어

**Files:** Create `snapdoc/src/edge.ts`, `snapdoc/src/quad.ts`, `snapdoc/test/detect.test.mjs`

**Interfaces:**
- Consumes: `Gray`
- Produces: `edge.ts`: `sobel(g: Gray): Gray`(그래디언트 크기), `threshold(g: Gray, ratio=0.25): Uint8Array`(max 기준 비율 임계)
- `quad.ts`: `findQuad(edges: Uint8Array, w, h): Quad | null` — 각 행/열 스캔으로 외곽 엣지 포인트 수집 → 좌상/우상/우하/좌하 극점 후보 → 볼록성·최소면적(전체의 10%)·직각성 검증, 스코어 = 면적 × 직각성(코너 코사인 페널티)

**Steps:** 실패 테스트(검은 배경에 기울어진 흰 사각형 합성 → findQuad 코너 오차 ±3px, 문서 없는 노이즈 이미지 → null) → 구현 → 통과 → 커밋 `feat(snapdoc): document quad detection (sobel + convex quad scoring)`

### Task 3: homography + warp-cpu (+테스트)

**Files:** Create `snapdoc/src/homography.ts`, `snapdoc/src/warp-cpu.ts`, `snapdoc/test/warp.test.mjs`

**Interfaces:**
- Produces: `homography.ts`: `computeH(src: Quad, dstW, dstH): Float64Array`(3x3, dst→src 역방향), 내부 8x8 가우스 소거
- `warp-cpu.ts`: `warpCpu(rgba: Uint8ClampedArray, srcW, srcH, H, dstW, dstH): Uint8ClampedArray` — 역방향 매핑 + bilinear
- 출력 크기 규칙: `dstW/dstH = quad 변 평균 길이`, maxSize(기본 1600) 초과 시 비율 축소 — index.ts에서 계산해 전달

**Steps:** 실패 테스트(단위 사각형 항등 H 검증, 기울어진 컬러 사각형 워핑 후 코너 픽셀 색 검증) → 구현 → 통과 → 커밋 `feat(snapdoc): homography and CPU perspective warp`

### Task 4: warp-gpu + index 공개 API

**Files:** Create `snapdoc/src/warp-gpu.ts`, `snapdoc/src/index.ts`

**Interfaces:**
- `warp-gpu.ts`: `warpGpuAvailable(): boolean`, `warpGpu(source: ImageBitmap|HTMLCanvasElement, H: Float64Array, dstW, dstH): Promise<HTMLCanvasElement>` — WGSL 프래그먼트에서 H 역변환 좌표로 textureSampleLevel(bilinear), 디바이스는 모듈 레벨 1회 초기화 캐시
- `index.ts`: 스펙의 공개 API 5개 (`detect/warp/quality/scan/highlight`) — CanvasImageSource→ImageData 변환, 360px 축소, 좌표 원본 스케일 복원, warp는 GPU 시도 후 실패 시 CPU 폴백, highlight는 rAF 루프에서 detect+코너 폴리곤 stroke, 반환 함수로 중단
- 브라우저 전용이라 node 테스트 없음 — Task 5 데모에서 실검증

**Steps:** 구현 → `npm run build` 통과(타입 에러 0) → 커밋 `feat(snapdoc): WebGPU warp and public API (detect/warp/quality/scan/highlight)`
